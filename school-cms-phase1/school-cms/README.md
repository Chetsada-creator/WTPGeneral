# เว็บไซต์ฝ่ายบริหารงานทั่วไป + ระบบ CMS (เฟส 1)

ระบบเว็บไซต์พร้อมระบบจัดการเนื้อหาแบบ Modular สำหรับฝ่ายบริหารงานทั่วไปของโรงเรียน
พัฒนาด้วย **PHP 8 + MySQL** ไม่ใช้ framework — ติดตั้งได้บนโฮสต์ติ้งทั่วไปหรือ XAMPP

## ความสามารถ (เฟส 1)

| ส่วน | รายละเอียด |
|---|---|
| ข่าว & ประกาศ | ร่าง/เผยแพร่, ตั้งเวลาเผยแพร่, ปักหมุด, ข่าวเด่น, ปก, แกลเลอรี, เอกสารแนบ, หมวดหมู่, นับผู้ชม |
| ดาวน์โหลดเอกสาร | อัปโหลด PDF/Word/Excel ฯลฯ, หมวดหมู่, นับจำนวนดาวน์โหลด |
| บุคลากร | รูปถ่าย, ตำแหน่ง, ฝ่ายงาน, ค้นหา/กรอง, ลากจัดลำดับ |
| ปฏิทินกิจกรรม | มุมมองรายเดือน + รายการ, 4 ประเภทกิจกรรม |
| หน้าเนื้อหา | สร้างหน้าใหม่ได้ไม่จำกัด ผูกกับเมนูได้ |
| **Menu Builder** | สร้างเมนูหลัก/เมนูย่อย เชื่อมกับโมดูล หน้าเนื้อหา หรือลิงก์ภายนอก ผ่าน dropdown ไม่ต้องแก้โค้ด |
| **Content Blocks** | บล็อก 5 ชนิด (แบนเนอร์ / กล่องประกาศ / ปุ่มลัด / การ์ดข้อมูล / HTML) วางได้ 5 ตำแหน่งบนเว็บ ลากจัดลำดับได้ |
| Quick Links | การ์ดลิงก์ด่วนหน้าแรก + ลิงก์ footer |
| ตั้งค่าโรงเรียน | ชื่อ โลโก้ สีธีม ฟอนต์ ที่อยู่ โซเชียล แผนที่ SEO — เปลี่ยนได้ทั้งเว็บทันที |
| Module Manager | เปิด/ปิดโมดูลได้ รองรับเพิ่มโมดูลใหม่ในอนาคต |
| ผู้ใช้ 5 ระดับ | super_admin / admin / editor / staff / viewer |
| อื่น ๆ | Dark/Light Mode, Responsive, นับผู้เข้าชม, Activity Log, CSRF protection |

## วิธีติดตั้ง

### บนโฮสต์ติ้ง (cPanel / DirectAdmin / Plesk)
1. อัปโหลดไฟล์ทั้งหมดขึ้นโฮสต์ (โฟลเดอร์ `public_html` หรือโฟลเดอร์ย่อย)
2. สร้างฐานข้อมูล MySQL + ผู้ใช้ฐานข้อมูล จากแผงควบคุมโฮสต์
3. เปิดเบราว์เซอร์ไปที่ `https://โดเมนของท่าน/install.php`
4. กรอกข้อมูลฐานข้อมูล และตั้งบัญชีผู้ดูแลระบบสูงสุด → กดติดตั้ง
5. **ลบไฟล์ `install.php` ออกจากโฮสต์ทันที**
6. เข้าระบบจัดการที่ `/admin/`

### บนเครื่องโรงเรียน (XAMPP)
1. คัดลอกโฟลเดอร์นี้ไปไว้ที่ `C:\xampp\htdocs\school`
2. เปิด XAMPP Control Panel → Start **Apache** และ **MySQL**
3. เข้า `http://localhost/phpmyadmin` → สร้างฐานข้อมูลชื่อ `school_cms` (collation `utf8mb4_unicode_ci`)
4. เปิด `http://localhost/school/install.php` แล้วทำตามขั้นตอน (ผู้ใช้ฐานข้อมูล `root` รหัสผ่านเว้นว่าง)
5. ลบ `install.php`

> ต้องการ PHP 8.0 ขึ้นไป และโฟลเดอร์โปรเจกต์ต้องเขียนไฟล์ได้ (สำหรับ `config.php` และ `uploads/`)

## โครงสร้างโปรเจกต์

```
school-cms/
├── index.php            หน้าเว็บสาธารณะ (front controller)
├── install.php          ตัวติดตั้ง (ลบทิ้งหลังติดตั้ง)
├── install.sql          โครงสร้างฐานข้อมูล + ข้อมูลตั้งต้น
├── config.php           สร้างอัตโนมัติตอนติดตั้ง
├── includes/            db, ฟังก์ชันกลาง, ระบบสิทธิ์
├── templates/           header / footer ของหน้าเว็บ
├── modules/             หน้าแสดงผลแต่ละโมดูล
├── admin/               ระบบจัดการ (เข้าที่ /admin/)
├── assets/              CSS / JS
└── uploads/             ไฟล์ที่อัปโหลด (สำรองข้อมูลโฟลเดอร์นี้ด้วย)
```

## ระดับสิทธิ์

| สิทธิ์ | ทำอะไรได้ |
|---|---|
| viewer | ดูแดชบอร์ด |
| staff | จัดการข่าว/ประกาศ/เอกสาร/บุคลากร/กิจกรรม (ลบไม่ได้) |
| editor | ทั้งหมดของ staff + ลบเนื้อหา + หน้าเนื้อหา + บล็อก + ลิงก์ + หมวดหมู่ |
| admin | ทั้งหมดของ editor + เมนู + โมดูล + ตั้งค่า + ผู้ใช้ (ยกเว้นบัญชี super_admin) |
| super_admin | ทุกอย่าง + บันทึกการใช้งาน (logs) |

## การสำรองข้อมูล

สำรอง 2 อย่างเป็นประจำ: **ฐานข้อมูล MySQL** (export จาก phpMyAdmin) และ **โฟลเดอร์ `uploads/`**

## แผนเฟสถัดไป

- เฟส 2: Page Builder แบบ block-based เต็มรูปแบบ, Media Library กลาง, ค้นหารวมทั้งเว็บ
